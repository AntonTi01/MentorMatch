# MentorMatch pgAdmin Container

## Назначение
Веб-интерфейс pgAdmin 4 для управления базой данных MentorMatch. Упрощает отладку и анализ данных, предоставляя GUI для подключения к контейнеру Postgres по внутренней сети Docker.【F:docker-compose.yml†L143-L155】

## Конфигурация
- Использует официальный образ `dpage/pgadmin4` и поднимается с именем `mentormatch-pgadmin`. Контейнер автоматически перезапускается при сбоях благодаря `restart: unless-stopped`.【F:docker-compose.yml†L143-L151】
- Доступен на хосте по адресу `http://localhost:${PGADMIN_PORT:-8080}` благодаря пробросу порта `80` контейнера на внешний порт, задаваемый переменной окружения (по умолчанию 8080).【F:docker-compose.yml†L147-L150】
- Учетные данные администратора (email и пароль) берутся из переменных `PGADMIN_EMAIL` и `PGADMIN_PASSWORD`, задаваемых в `.env` или окружении Compose. Это обеспечивает контролируемый доступ к веб-интерфейсу.【F:docker-compose.yml†L145-L150】
- Том `pgadmin_data` монтируется в `/var/lib/pgadmin`, сохраняя историю подключений, настройки и серверные профили между перезапусками контейнера.【F:docker-compose.yml†L150-L152】

## Использование
- После запуска `docker-compose up` перейдите на опубликованный порт pgAdmin и добавьте подключение к серверу `mentormatch-postgres`, используя те же учетные данные Postgres, что и в `.env`.
- Контейнер зависит от `postgres`, поэтому доступ к базе данных становится возможным, когда healthcheck Postgres отмечает готовность (контейнер pgAdmin стартует после Postgres).【F:docker-compose.yml†L151-L152】
